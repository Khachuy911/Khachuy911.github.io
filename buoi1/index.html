1) git config

Tác dụng để set user name và email của bạn trong main configuration file. Cách sài : Để kiểm tra tên và kiểu email trong cấu hình dùng git config --global user.name và git config --global user.email . Để set email hoặc tên mới git config --global user.name = "Khachuy911" và git config --global user.email = "khachuy469@gmail.com".

2) git init

Tác dụng: Khởi tạo 1 git repository 1 project mới hoặc đã có.
Cách xài : git init trong thư mục gốc của dự án.

3) git clone

Tác dụng : Copy 1 git repository từ remote source.
Cách xài : git clone <: clone git url:>

4) git status

Tác dụng: đẻ check trạng thái của những file bạn đã thay đổi trong thư mục làm việc. VD: Tất các thay đỏi cuối cùng từ lần commit cuối cùng
Cách xài: git status trong thư mục làm việc

5)git add

Tác dụng: Thêm thay đổi đến stage/index trong thư mục làm việc.
Cách xài :git add.

6) git commit

Tác dụng:commit nghĩa là một action để Git lưu lại một snapshot cuả các sự thay đổi trong thư mục làm việc. Và các tập tin , thư mục được thay đổi đã phải nằm trong stagin Area.Mỗi lần commit nó sec được lưu lại lịch cảu code kèm theo tên và địa chỉ email của người commit. Ngoài ra trong git bạn cũng có thể khôi phục lại tập tin trong lịch sử commit của nó để chia cho branch khác, vì vậy bajnsex dễ dàng khôi phục lại các thay đổi trước đó.
Cách dùng : git commit -m "note lại những thay đổi".

7)git push/git pull

Tác dụng : Push hoặc pull các thay đổi dến remote.Nếu bạn đã added và commited các thay đổi và bạn muốn đẩy nó lên hoặc remote cảu bạn đã update và bạn apply tất cả thay đỏi đó trên code của mình

8) git remote

Tác dụng : để check remote/source bạn có hoặc add thêm remote.
Cách dùng: git remote để kiểm tra và liệt kê . Và git remote add <:remote_url:>để thêm

9)git add

Tác dụng: Để đưa một tập tin vào staging area
Cách dùng: git add tên_file hoặc muốn thêm hết file cảu thư mục thì git add all
